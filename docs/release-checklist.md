# basis-gateway Release Checklist

This checklist is used for every tagged release of `basis-gateway`. Complete each section before tagging. For v0.1.0, items marked **[v0.1 resolved]** were completed as part of the release readiness review.

---

## 1. Repository hygiene

- [ ] `LICENSE` is present and matches the rest of the BASIS ecosystem (Apache 2.0)
- [ ] `SECURITY.md` is present and describes the current security model and reporting path
- [ ] `CHANGELOG.md` has an entry for this release
- [ ] `README.md` accurately describes what is implemented (not aspirational)
- [ ] `.env.example` uses placeholder values only — no real credentials, tokens, or issuer URLs
- [ ] No secrets, tokens, or credentials committed to the repository (`git log` check)
- [ ] All "phase N" or internal build-tracking language removed from user-facing docs and source docstrings
- [ ] `pyproject.toml` version field matches the release tag

**[v0.1 resolved]** LICENSE, SECURITY.md, CHANGELOG.md, .github/ created. README verified. No secrets found. Phase language removed.

---

## 2. Documentation completeness

- [ ] `README.md` includes: feature list, setup instructions, environment variable reference, API reference, architecture position
- [ ] `docs/release-readiness.md` accurately describes scope, out-of-scope items, known limitations, and architecture invariants
- [ ] `docs/audit-model.md` open questions section is current; resolved items marked *(resolved)*
- [ ] `docs/troubleshooting.md` covers the most likely failure modes
- [ ] `docs/releases/vX.Y.Z.md` release notes written and reviewed
- [ ] Links in docs verified (no broken relative paths)

**[v0.1 resolved]** All docs reviewed, updated, and verified. Pre-evaluation audit coverage question resolved and documented.

---

## 3. Security review

- [ ] Authentication path: token verification covers expected algorithm set; `alg=none` rejected
- [ ] Identity normalization: subject identity derived from token only; request body cannot supply `subject_id` or `subject_roles`
- [ ] Audit evidence: all pre-evaluation failure paths emit a gateway event; verified by tracing `evaluate()` handler
- [ ] Correlation ID: generated by gateway unconditionally; caller-supplied values ignored
- [ ] No raw token material in logs, responses, or audit records (verified in `auth/oidc.py`, `audit/gateway_events.py`)
- [ ] `.env.example` — no real credentials
- [ ] `policies/default.json` — no sensitive data

**[v0.1 resolved]** All paths verified. No secrets found. See `docs/audit-model.md` §9 for the latent `RequestValidationError` gap (currently unreachable).

---

## 4. Dependency review

- [ ] All runtime dependencies at or above minimum versions in `pyproject.toml`
- [ ] No dependency with a known CVE at the pinned or minimum version
- [ ] `basis-core` minimum version pin matches the public API surface in use
- [ ] Dev dependencies verified (pytest, ruff, mypy, cryptography)

**[v0.1 resolved]** All installed versions well above minimums. No CVEs identified at current versions.

---

## 5. Test suite

- [ ] `pytest` passes with no failures or errors
- [ ] `ruff check src tests` passes with no violations
- [ ] `ruff format --check src tests` passes
- [ ] `mypy src` passes with no errors
- [ ] Test count is consistent with what the README references

---

## 6. API contract verification

- [ ] `EvaluateResponse`: `correlation_id` always present; `policy_version` omitted when `POLICY_VERSION` not configured (verified by `exclude_none=True` and `test_policy_version_null_when_not_configured`)
- [ ] `ErrorResponse`: `correlation_id` present on all error paths; `message` omitted when null
- [ ] `GET /ready` component list matches the components registered in `readiness.py` and documented in `README.md`
- [ ] `X-Correlation-ID` header present on all responses including 400, 401, 403, 500, 503

**[v0.1 resolved]** API examples in README verified against test suite and `exclude_none=True` behavior.

---

## 7. Release tagging

- [ ] All checklist items above complete
- [ ] `docs/releases/vX.Y.Z.md` written and reviewed
- [ ] `docs/release-candidate-assessment.md` written (for major releases or release candidates)
- [ ] Tag created: `vX.Y.Z`
- [ ] GitHub Release created from tag with release notes

---
